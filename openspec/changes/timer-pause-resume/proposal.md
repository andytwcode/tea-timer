## Why

目前計時過程中的按鈕為禁用狀態，使用者無法暫停正在進行的計時。這在需要短暫中斷（接電話、開門、處理突發狀況）時不方便，只能選擇等待計時完成或重置回初始狀態，造成使用體驗不佳。

## What Changes

- 計時中的主按鈕從「禁用」改為「可點擊的暫停功能」
- 新增「已暫停」狀態，保留當前剩餘時間
- 暫停後可透過相同按鈕繼續計時
- 暫停期間時間輸入框保持鎖定（簡單暫停策略）
- 連續沖泡模式下，暫停僅影響當前泡，不改變基準時間或後續泡的預計時間

## Capabilities

### New Capabilities

無新增功能模組。

### Modified Capabilities

- `timer-control`: 修改開始按鈕行為，從「倒數中禁用」改為「提供暫停/繼續切換」；新增暫停狀態的管理和按鈕文字邏輯。

## Impact

**程式碼**:
- `src/App.vue` - 狀態管理（新增暫停狀態變數）
- `src/App.vue` - 計時器控制邏輯（暫停時清除 interval，繼續時重新啟動）
- `src/App.vue` - 按鈕邏輯（主按鈕在計時中顯示「暫停」，暫停中顯示「繼續」）

**不影響**:
- 通知系統（通知時機不變，仍在 remainingSeconds 達到 0 時觸發）
- PWA manifest 配置
- 其他 UI 元件
